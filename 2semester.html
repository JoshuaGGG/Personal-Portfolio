EECU Budget Calculator

In the start of the second semester we had a coordinator from EECU and we had learned all about finances and how each career applies to a budget within your net income. The main idea was to create a budget calculator that calculates the career income after taxes and other spending variables using javaScript. The design of this website was wireframed with the use of figma. A simple design was done on paper but over time the design of the wireframe evolved with the addition of programming. HTML, CSS and JavaScript were the main building blocks for the entirety of this project. Lucid chart was in correlation with what the process was going to be for functionality or the JavaScript of the website. I decided to make a list of all of the careers and money values associated within their careers. Each career has a data value and once clicked on each of the variables on the for loop will calculate the career income after each property. The outcome of the calculation would be displayed on the second career box and rounded to two figures. An array for every available career had to be defined for the amount of careers. A for loop added the event listener for each element in the career list. If an element in the career list is clicked then the data values begin to calculate the taxes and income correlated information on the career. JavaScript updates each time to the career selected in the career list and displays it as inner HTML. The challenge that I faced the most was getting used to JavaScript I wasn’t as familiar with the language but I quickly learned through my mistakes. It was a great learning experience to do research from w3 schools tutorials and general coursera information.

Loteria

The second project of the second semester was to make a digital boardgame using HTML, css and JavaScript. This was a team project of two people working together to finish the project. Researching what game we would make was the initial start of the project and we chose Loteria a mexican traditional bingo game. The original instructions for the game were to fill the entire 4x4 board with tokens from a card shuffled each time. There was modifications that we had to make over the original ideas in order to make it more fast paced. Modifications like expanding the board to a 5x5 and adding two more cards to the deck was the solution. Like any project a wireframe or overall design had to be made in figma. We wanted a clear and crispy layout and decided to have  the instructions on the first page and the next page to be the selection of the boards. Once you selected a board the game would start and hide all of the remaining elements. The illustrious design that we wanted was mostly included in the card images itself and the rgb keyframes that were added made it look like an actual game environment. The completion of the preparation  mean’t that the next step was to program the entirety of the website in Visual Studio Code. HTML made the overall layout or backbone of the website and CSS helped to style from the position to the colors. Most of the trouble was gaining traction in JavaScript where some of the code was not working properly because we weren’t defining a certain property or general code was missing something. We had to understand how JavaScript understands assignments and rules set for the creation of generating boards, card display and drag distribution of a token. Generating Boards involved constant for loops in order to process the randomly generated boards and setting values less than three to have similar card sets in the board.  For the card display a for loop iterated over the last index to the beginning of the index and swapped the initial index to the  upcoming randomly generated index. This was an array of a total of 55 cards with the addition of the takeaway and wildcard. The distribution of a token was definitely a challenge as we didn’t know how to make an image dragable and able to be put in front of another image. We found a whole JavaScript tutorial online of drag and drop. We had to set the generated board images as background image and the drag and drop as functions to fire the dragable elements as well as calling the event handler to make the element a valid drop target. Much was learned from this project in general programming and collaboration of the work with a classmate.
